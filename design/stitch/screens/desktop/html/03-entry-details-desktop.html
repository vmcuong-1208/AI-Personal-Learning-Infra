Core Principles

BullMQ is a Node.js message queue library that leverages Redis for persistence and atomicity. Unlike simpler task runners, BullMQ is built for enterprise-grade scalability, handling features like stalled job detection, parent-child job dependencies, and rate limiting natively.

The concurrency model relies on Worker instances that poll Redis. Each worker can process jobs in parallel depending on the configuration. Key takeaway: Atomic operations in Redis ensure that a job is never picked up by two workers simultaneously, even in a high-traffic distributed system.

"Observation: In production, we noticed that setting concurrency too high on I/O bound tasks led to connection exhaustion. The sweet spot for our Node clusters was 20 workers per instance."
