# Source Structure

This folder is reserved for the application implementation.

Recommended organization:

- `app`: routing, app shell wiring, providers.
- `components/layout`: app layout, sidebar, mobile navigation.
- `components/ui`: reusable UI primitives.
- `features`: screen and domain-specific modules.
- `data/mock`: local mock data until API integration exists.
- `lib`: shared utilities.
- `styles`: global styles and design tokens.
